System Architecture

The AI-Based Memory Forecaster collects real-time memory telemetry, engineers time-series features, trains Random Forest and LSTM models to predict future memory demand, and feeds those forecasts into a proactive decision engine.

Feature Engineering

features.py transforms raw memory logs into an ML-ready matrix with lag features, rolling windows, rates of change, and process context.

Lag Features

Previous timestep memory values

Rolling Window
mem_roll_mean mem_roll_std mem_roll_min mem_roll_max

Moving statistics (w=30 samples)

Rate of Change
mem_delta time_delta rate_mb_per_sec

Memory velocity between samples

Top-5 process RSS (resident set size)

Decision Engine & Simulation

The decision engine maps forecasted memory to proactive OS actions. Thresholds are configurable and evaluated row-by-row on the feature matrix.

Decision Thresholds
0% — Normal (none) 85% — Swap Early 95% — Throttle OOM 100%

Additional rule: if forecast - current > 200 MB → pre-allocate

ConditionActionMeaning
Forecast > 95% RAM throttle_oom Imminent OOM — throttle/kill low-priority
Forecast > 85% RAM swap_early High pressure — early swap-out
Forecast − Current > 200 MB prealloc Spike detected — pre-allocate
Otherwise none Normal range, no action needed
